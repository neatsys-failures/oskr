Detailed Description

template<typename Transport>
class oscar::TransportMulticastReceiver< Transport >

Although theoretically a multicast receiver is not required to be addressable if it does not listen to any unicast, the situation not present for known protocols (or can be easily workaround-ed by assigning a dummy address), and we try to avoid multi-inheritance here.
